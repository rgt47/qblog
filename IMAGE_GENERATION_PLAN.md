# Blog Image Generation Plan

Process for sourcing or generating the hero + 3 ambiance images required by
the post-39 (analysis) and post-47 (setup) templates.

All images live in `posts/<NN-name>/<name>/media/images/` and must be
attributed in `media/images/README.md` in that same directory.

## Slot specifications

| Slot           | Width | Post 39 (analysis)               | Post 47 (setup)                    |
|----------------|-------|----------------------------------|------------------------------------|
| Hero           | 80%   | Topic-evocative photo            | Tool logo or workspace composite   |
| Ambiance 1     | 100%  | Atmospheric, after Objectives    | Workspace shot (desk, hardware)    |
| Ambiance 2     | 100%  | After first analysis figure      | Terminal or editor in action       |
| Ambiance 3     | 100%  | Before Lessons Learnt            | Workflow scene (tool in use)       |

Analysis posts emphasise mood and topic. Setup posts tell a
'before, during, after' install story, so two of the four slots should be
your own screenshots.

## Sourcing strategy by post type

### Post 39 (analysis) sourcing mix

- 1 hero from Unsplash, Pexels, or AI generation
- 2 ambiance images from the same stock or AI source for visual cohesion
- 1 ambiance image generated in R from the post's own data, acting as a
  teaser of the analysis (consider `ggplot2`, `aRtsy`, `generativeart`)

Rationale: analysis posts should feel editorial. Cohesive photographic or
illustrative styling carries the reader; the R-generated piece signals the
post's data-driven content without competing with the figures themselves.

### Post 47 (setup) sourcing mix

- 1 hero from AI generation (DALL-E 3, Midjourney) or Unsplash workspace
- 1 ambiance from Unsplash workspace category (desk, hardware, coffee)
- 2 ambiance from your own screenshots (terminal and editor in use)

Rationale: setup posts gain credibility from authentic captures. Stock
imagery alone reads as generic. Mix roughly 50% own screenshots, 25%
stock, 25% AI.

## Step-by-step workflow

### Step 1: plan the four shots (5 minutes)

Before opening any tool, write the four captions you want a reader to
intuit from the images. For an `ellmer` setup post:

- Hero: 'an R user composing prompts as easily as code'
- Ambiance 1: 'clean workspace ready for a focused session'
- Ambiance 2: 'live terminal showing ellmer::chat_claude()'
- Ambiance 3: 'split-screen editor with chat output'

This 5-minute exercise prevents wandering through stock photo sites for
an hour with no anchor.

### Step 2: source or generate

Free stock sources (no attribution legally required, but include it):

- Unsplash, https://unsplash.com
- Pexels, https://pexels.com
- Pixabay, https://pixabay.com

AI generation prompt template (analysis post):

```
Editorial-style hero image for a blog post about [topic]. [Mood, e.g.,
contemplative, energetic, technical]. 16:9 aspect ratio, no text, muted
palette, photographic realism.
```

AI generation prompt template (setup post):

```
Minimal flat-lay of a developer workspace at dusk, MacBook showing a
terminal, muted earth tones, 16:9, no text, editorial photography style.
```

Own screenshots on macOS:

- `Cmd-Shift-4` then `Space` for window captures
- `Cmd-Shift-5` for region or timed captures
- Clean the terminal first: `clear`, hide secrets, set font to 14pt or
  larger, prefer a calm colour scheme (Solarized, Nord)

R-generated hero or teaser:

```r
library(ggplot2)
p <- ggplot(your_data, aes(x, y, colour = group)) +
  geom_point(size = 3, alpha = 0.7) +
  scale_colour_viridis_d() +
  theme_minimal(base_size = 14) +
  theme(legend.position = 'none',
        axis.title = element_blank(),
        axis.text = element_blank())
ggsave('media/images/hero-teaser.png', p,
       width = 16, height = 9, units = 'in', dpi = 150)
```

For hex-sticker style heroes (R package setup posts):

```r
library(hexSticker)
sticker('logo.png', package = 'pkgname',
        s_x = 1, s_y = 0.75, s_width = 0.6,
        h_fill = '#1a1a2e', h_color = '#e94560',
        filename = 'media/images/setup-hero.png')
```

### Step 3: process and store

Resize, strip EXIF, and standardise quality:

```bash
cd posts/<NN-name>/<name>/media/images/
magick hero-raw.jpg   -resize 1600x -strip -quality 85 hero.jpg
magick ambiance-01.jpg -resize 1920x -strip -quality 85 ambiance-01.jpg
magick ambiance-02.png -resize 1920x -strip -quality 90 ambiance-02.jpg
magick ambiance-03.png -resize 1920x -strip -quality 90 ambiance-03.jpg
```

Naming convention:

- `hero.jpg` (or `hero.png` if generated by R or hexSticker)
- `ambiance-01.jpg`, `ambiance-02.jpg`, `ambiance-03.jpg`

Match the names to the YAML `image:` field and the `![](media/images/...)`
references in `index.qmd`.

### Step 4: attribute every image

Append to `media/images/README.md`:

```
- hero.jpg          — Photo by [Name] on Unsplash, https://unsplash.com/photos/XXXX
- ambiance-01.jpg   — Photo by [Name] on Pexels, https://pexels.com/photo/XXXX
- ambiance-02.jpg   — Own screenshot, 2026-04-14, iTerm2 + Fira Code 14pt
- ambiance-03.png   — Generated with DALL-E 3, prompt: '...'
```

For R-generated images, point at the script:

```
- hero-teaser.png   — R-generated, see analysis/scripts/03_generate_figures.R
```

### Step 5: verify in the rendered post

```bash
cd ~/Dropbox/prj/qblog
quarto render 'posts/<NN-name>/<name>/index.qmd' --to html
quarto preview
```

Check the blog listing card uses the hero, and that the three ambiance
images break up the body text without crowding the analysis figures.

## AI image generation: tool selection and prompts

This section expands Step 2 with practical guidance for choosing an AI
image generator and writing prompts that work for the editorial style our
templates expect.

### Tool comparison

| Tool                | Strength                                            | Weakness                                              | Cost                              | Best for                                       |
|---------------------|-----------------------------------------------------|-------------------------------------------------------|-----------------------------------|------------------------------------------------|
| Midjourney v7       | Best photorealism + composition; cinematic         | Discord/web UI; steeper curve; aspect via `--ar`      | $10/mo basic                      | When you want the image to look 'expensive'    |
| DALL-E 3            | Best instruction-following; conversational         | Aesthetic 'competent'; gibberish text on screens      | Free via Bing; $20/mo ChatGPT Plus | Complex/specific prompts                       |
| Google Imagen 3     | Strong photorealism; best at typography            | Less editorial polish than MJ                         | Free in Gemini app; $20/mo Advanced; Vertex AI ~$0.04/img | Free option that does not look generic |
| Flux 1.1 Pro        | Approaches MJ quality, much cheaper                | No consumer UI; via Replicate / fal.ai                | ~$0.04/img on fal.ai              | Bulk or programmatic workflows                 |
| Adobe Firefly       | Commercial-safe (licensed training)                | Aesthetically conservative                            | Bundled in Adobe CC               | When licence cleanliness matters               |
| Stable Diffusion    | Free, fully controllable, runs on Apple Silicon    | Hours of setup; worse out-of-box quality              | Free + your time                  | Hobbyist tinkering                             |

### Recommended path for one-off blog heroes

1. Try **Google Imagen 3** in [gemini.google.com](https://gemini.google.com)
   first. Free, competitive quality, no commitment.
2. If the result feels generic, sign up for **Midjourney** for one month
   ($10), generate hero candidates for several upcoming posts at once,
   cancel after.
3. Reserve **DALL-E 3 via ChatGPT** only if you already pay for ChatGPT
   Plus for other reasons.

### Aspect ratio cheat sheet

Hero images on this blog use 80% width display, ideally 16:9 source.

- **DALL-E 3**: only `1024x1024`, `1792x1024`, `1024x1792`. Use
  `1792x1024` for hero. Specify in prompt: 'Generate a 1792x1024 image...'
- **Midjourney**: use `--ar 16:9` flag. Combine with `--style raw` to
  remove MJ's default stylisation for editorial realism.
- **Imagen 3 / Gemini**: state '16:9 landscape' explicitly in the prompt.
  Imagen honours this reliably.

### Prompt structure (works across all tools)

Three things matter most. Order them in this sequence:

1. **Subject + composition**: 'a 14-inch laptop on a walnut desk,
   three-quarter view'
2. **Lighting + palette**: 'late-afternoon natural light, muted
   earth-tone palette'
3. **Style**: 'editorial photography, 35mm lens, shallow depth of field,
   photographic realism'

Avoid: text on screens (all models hallucinate gibberish), specific brand
logos (filtered or distorted), more than ~3 distinct objects in frame,
people unless central to the concept.

### Worked example: post 22 hero (AWS CLI provisioning)

Same conceptual prompt, three syntaxes.

#### Gemini (Imagen 3) — copy-paste verbatim

```
Generate a 16:9 landscape image with the following description:

Editorial photograph, three-quarter view of a 14-inch silver
laptop on a warm walnut desk. The screen displays a terminal
window with green monospace text on a dark background — the text
is intentionally illegible, treated as soft blur, with no
readable words. Behind the laptop, slightly out of focus, a
server rack with rows of cyan LED indicator lights stretches
into shallow depth-of-field bokeh. Late-afternoon natural
sunlight enters from the left at a low angle. Muted earth-tone
palette (warm browns, soft blacks, cyan accents only on the
LEDs). Style: editorial photography, 35mm lens aesthetic,
photographic realism, no text overlays, no logos, no people.
```

#### DALL-E 3 (ChatGPT Plus or Bing) — copy-paste verbatim

```
Generate a 1792x1024 image with the following description:

Editorial photograph, three-quarter view of a 14-inch MacBook
Pro on a warm walnut desk. The screen displays a terminal window
with green monospace text on a dark background (text is
intentionally illegible blur, no readable words). Behind the
laptop, slightly out of focus, a server rack with rows of cyan
LED indicator lights stretches into shallow depth-of-field
bokeh. Late-afternoon natural light from the left, muted
earth-tone palette (warm browns, soft blacks, cyan accents).
Photographic realism, 35mm lens aesthetic, no text overlays, no
logos.
```

#### Midjourney — Discord/web

```
14-inch MacBook on warm walnut desk, three-quarter view,
terminal window with abstract green text (illegible), softly
out-of-focus server rack with cyan LEDs in background,
late-afternoon natural light, muted earth tones, editorial
photography, 35mm lens, shallow depth of field, photographic
realism --ar 16:9 --style raw --v 7
```

### Iteration loop (typical session)

Three rounds is normal. Refine conversationally:

```
You: [paste prompt above]
AI: [generates 1-4 images]
You: I like the composition but the server rack is too prominent.
     Move it further back, more bokeh, less cyan.
AI: [regenerates]
You: Closer. Now warm the colour grade slightly and lower the
     laptop screen brightness so it is not the focal point.
```

### Common failure modes and fixes

| Problem                       | Fix                                                       |
|-------------------------------|-----------------------------------------------------------|
| Gibberish text on screen      | Add 'text intentionally illegible / blurred / out of focus' |
| Too cluttered                 | Drop adjectives; aim for ≤3 distinct objects in frame    |
| Wrong aspect                  | Restate aspect explicitly ('1792x1024' or '16:9 landscape') |
| Generates square anyway       | Ask again: 'use the wide 16:9 format'                     |
| Looks like generic stock      | Add 'editorial photography, 35mm lens, shallow depth of field' |
| Brand-filtered (no Apple logo) | Use 'generic silver laptop' instead of 'MacBook'          |
| Adds people unprompted        | Add 'no people, no hands visible'                         |

### After the download

```bash
cd posts/<NN-name>/<name>/media/images/
mv ~/Downloads/dalle-result.png hero-raw.png
magick hero-raw.png -resize 1600x -strip -quality 85 hero.jpg
rm hero-raw.png
```

Then append to `media/images/README.md`:

```
- hero.jpg — Generated with [Imagen 3 via Gemini | DALL-E 3 via
  ChatGPT | Midjourney v7], YYYY-MM-DD.
  Prompt: '[paste full prompt verbatim]'
  Original: [WxH] PNG; downsized to 1600px wide JPEG q85.
```

Pasting the full prompt is important — it lets you regenerate or
variant-iterate later without rewriting from memory.

## Quick-decision rules of thumb

- If the post's topic has a strong visual concept (a place, an object, a
  metaphor): use Unsplash or AI for the hero.
- If the post is about an R package or CLI tool: own screenshots win on
  credibility for ambiance 2 and 3.
- If the post centres on data: include at least one R-generated image,
  even if it duplicates an analysis figure stylistically.
- Never mix four sharply different styles (photo + cartoon + screenshot +
  abstract) in one post; pick a dominant aesthetic.

## Helper script idea

A small R helper could automate steps 3 and 4:

```r
prepare_images(
  hero          = 'raw/hero.png',
  ambiance      = c('raw/desk.jpg', 'screen-term.png', 'screen-editor.png'),
  attributions  = c(
    hero          = 'DALL-E 3, prompt: ...',
    ambiance_01   = 'Photo by X on Unsplash, URL',
    ambiance_02   = 'Own screenshot, 2026-04-14',
    ambiance_03   = 'Own screencap, VS Code'
  ),
  post_dir      = 'posts/47-templatesetup/templatesetup'
)
```

This would resize, strip EXIF, rename to the convention, and append the
attribution block to `media/images/README.md`. Worth building once 5+
posts have followed the manual workflow and the friction is felt.
